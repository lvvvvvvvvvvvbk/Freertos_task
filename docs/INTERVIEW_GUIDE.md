# 面试讲解提纲（STM32F407 + FreeRTOS 直流充电桩）

## 1. RTOS 架构

- **任务划分**：安全保护、CAN接收、状态机、电表采集、云端上报、OTA、日志。
- **优先级策略**：保护与通信接收优先；云端和日志低优先。
- **反转处理**：共享资源使用互斥锁 + 优先级继承；ISR 使用任务通知减少开销。

## 2. 状态机

状态：`IDLE -> GUN -> INSULATION -> HANDSHAKE -> PARAM -> CHARGING -> FINISH`，异常进入 `FAULT`。

- 事件源：插枪 GPIO、绝缘检测 ADC、CAN 握手、刷卡鉴权、急停输入。
- 异常恢复：可恢复故障回到 `FINISH`/`IDLE`；不可恢复需人工复位。

## 3. 通信与协议

- **CAN + GB/T 27930**：报文接收后映射成内部事件。
- **RS485 + DL/T 645**：轮询电表，校验失败记录原始帧并重试。
- **MQTT/HTTP**：上报采用队列与限流，断网自动重连。

## 4. OTA（A/B 分区）

- 分片校验 -> 整包校验 -> 写后读回校验。
- 仅当全部通过才写入 pending 槽标记。
- 若新镜像启动失败，Bootloader 回滚到旧分区。

## 5. 调试证据链

- 统一日志格式：`[ts][task][state][event][rc]`
- 通信问题定位顺序：物理层（错误帧/终端电阻）-> 位时序 -> 软件队列。
- 时延测量：ISR 入口/出口翻转 GPIO，用示波器统计最大值和 P99。
